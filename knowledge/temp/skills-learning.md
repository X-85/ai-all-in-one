# Skills 学习笔记

## 1. Skills 是什么

**Skills（技能）** 是预定义的可复用任务模块，告诉 AI 在特定场景下该怎么做、提供什么能力。

**核心作用：**
- 🎯 专用能力：让 AI 在特定任务上更专业
- 🔌 即插即用：通过 `/技能名` 触发
- 📦 知识沉淀：把复杂工作流封装成一条命令

## 2. Skills 调用方式

- **触发**：对话中输入 `/技能名`（如 `/docx`）
- **过程**：AI 识别 → 加载技能指令 → 执行 → 返回结果
- **耗时**：几秒内完成

## 3. AI 与 Agent 的区别

**可以理解为 Agent，但有差异：**
- **Agent**：自主决策、连续执行多步任务，有自己的任务循环
- **AI 助手**：偏向对话式辅助，"被叫醒就干活"模式

**本质差别**：Agent 更强调自主性

## 4. Agent 与 Skills 的定位

- **Agent**：解决"怎么做"——自主拆解、连续执行多步任务
- **Skills**：解决"做什么"——专业领域标准化处理

## 5. Agent vs Plan 模式

- **Plan 模式**：拆解任务 → 给你方案 → 等你确认后执行（人在回路）
- **Agent**：拆解后自己跑完整个流程，人在关键节点介入

**一句话**：Plan 是"先给你看方案再干活"，Agent 是"直接干完再汇报"

## 6. 主 Agent vs 子 Agent

- **主 Agent**：任务执行者，有完整决策权，能调度工具和子任务
- **子 Agent**：被主 Agent 调用的专项执行单元，只负责分配的具体子任务

**关系**：主 Agent = 项目经理，子 Agent = 干活的小组成员

## 7. 子 Agent vs Skills

- **Skills**：静态指令集，加载后告诉 AI"按这个流程做"，没有自主决策
- **子 Agent**：独立执行体，能自主思考、调用工具、处理子任务

**区别**：Skills 是说明书，子 Agent 是能干活的临时工

## 8. 通用调用关系

- **主 Agent 和子 Agent 都可以调用 Skills**
- **Skills 内部一般不嵌套调用其他 Skills**（避免循环和复杂度）

## 9. Skills 的构成

Skills 通常包括：
- 专用工具（如 PDF 处理库、DOCX 操作库）
- 专业指令（该领域的处理流程、规范）
- 预设参数和模板

**核心**：把"领域知识 + 专用工具 + 标准流程"打包在一起

## 10. Skills 标准格式

通过 **SKILL.md** 文件定义：

```markdown
---
name: pdf
description: 专业 PDF 工具集...
---

# PDF 处理技能

## 适用场景
- 生成报告
- 提取内容

## 使用方法
输入 `/pdf` 触发

## 实现步骤
1. 判断任务类型
2. 调用对应工具
3. 返回结果
```

### 规范要点
1. **YAML 头**：必填 `name` 和 `description`
2. **描述清晰**：description 决定 AI 何时自动调用
3. **流程明确**：步骤化、可执行
4. **存放位置**：固定目录，AI 能自动发现

## 11. 子 Agent 自主执行保证

1. **明确的子任务描述** — 清晰的目标和边界
2. **工具权限** — 能调用所需工具
3. **上下文隔离** — 独立上下文，不被主任务干扰
4. **结果回传** — 完成后把结果交给主 Agent

**核心**：主 Agent 负责"拆得对"，子 Agent 负责"做完它"

## 12. 输出格式

- **Skills** 和 **子 Agent 输出** 都是 Markdown 文档
- **细微差异**：
  - Skills：固定结构（YAML 头 + 流程说明），是配置文档
  - 子 Agent 输出：是任务结果报告，结构自由
- **规范**：给人/AI 看的 Markdown，核心是清晰

---

## 总结

| 概念 | 一句话定位 |
|------|-----------|
| Skills | 专业领域的标准化处理流程（说明书） |
| Agent | 自主执行任务的实体（能干活的人） |
| 子 Agent | 被调用的专项执行单元（小组员） |
| Plan 模式 | 先确认后执行的规划方式 |
